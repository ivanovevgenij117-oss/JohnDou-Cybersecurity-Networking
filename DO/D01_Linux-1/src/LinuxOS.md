  ## Part 5. Использование команды sudo  

Команда sudo (от англ. superuser do) в операционных системах Unix-подобных (таких как Linux, macOS и BSD) позволяет пользователям выполнять команды с привилегиями суперпользователя (root). Это позволяет обычным пользователям выполнять административные задачи без постоянного входа в систему с учетной записью root.

![меняем порт](Images/43.png)  
Проверяем порт:  
![проверяем порт](Images/44.png)  

* Используя команду ps, покажи наличие процесса sshd.  

Используем команду: ps -fL -C sshd.  
ps - показывает состояние процессов  
-f выводит максимум доступных данных, например, количество потоков  
-L отображает потоки процессов в колонках LWP(легковесный процесс) и NLWP(число легковесных процессов)  
-С выбирает процессы по имени команды.  
![смотрим процесс ssh](Images/45.png)  

Перезагружаем систему.  

Вводим команду netstat -tan.  
![netstat -tan](Images/46.png)  
netstat - показывает содержимое различных структур данных, связанных с сетью, в различных форматах в зависимости от указанных опций.  
-tan: вывод всех активных подключений TCP и прослушиваемых компьютером портов TCP и UDP + показывает сетевые адреса как числа.  

State: Listen - ожидает входящих соединений.  
Proto: протокол, используемый сокетом.  
Recv-Q: количество байтов, не скопированных пользовательской программой, подключенной к этому сокету.  
Send-Q: количество неподтвержденных байтов удаленного хоста.  
Local address: локальный адрес (имя локального хоста) и номер порта сокета.  
Foreign address: удаленный адрес (имя удаленного хоста) и номер порта сокета.  
0.0.0.0 - это немаршрутизируемый адрес IPv4, который можно использовать в разных целях, в основном, в качестве адреса по умолчанию или адреса-заполнителя. Несмотря на то, что адрес 0.0.0.0 может использоваться в компьютерных сетях, он не является адресом какого-либо устройства.

  ![вывод утилиты top](Images/47.png)  
Uptime - 26 минут.  
Количество авторизованных пользователей - 1.  
Средняя загрузка системы - (0.00, 0.00, 0.00).  
Загрузка процессора:  
Пользовательские процессы занимают 0% от общего времени работы процессора.  
Системные процессы потребляют 0%.  
Процессор простаивает 100% времени.  
Остальное время (0%) уходит на другие незначительные операции.  
Загрузка памяти:  
Общее количество памяти (total): 8.9 Общий объем физической оперативной памяти в гигабайтах.  
Свободная память (free):  8.3 Количество свободной оперативной памяти, доступной для новых процессов в гигабайтах.  
Используемая память (used): 0.2 Объем оперативной памяти, занятый процессами в гигабайтах.  
Буфер и кэш (buff/cache): Память, используемая операционной системой для буферизации и кеширования данных. Этот объем памяти может быть освобожден при необходимости для новых процессов.  
Доступная память (avail Mem): Оценочное количество оперативной памяти, доступное для нового процесса без освобождения дополнительной памяти.  
PID-1 процесса занимающего больше всего памяти. Загрузка процессорного времени около 0%.
